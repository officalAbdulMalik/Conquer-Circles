───────────────────────────────────────────────────────────────────────────────
CRITICAL FINDINGS
───────────────────────────────────────────────────────────────────────────────

PROBLEM:
  The app calls claim_or_attack_tile() which is for HEX TILES,
  but the game uses TERRITORIES (polygon-based system).

IMPACT: 11 Critical Gaps
  ✗ Wrong table (hex_tiles vs territories)
  ✗ No speed validation (easy to cheat)
  ✗ Wrong energy model (fixed 10 damage vs full energy)
  ✗ Broken protection logic (shields bypass-able)
  ✗ Wrong capture logic (doesn't account for energy)
  ✗ Cooldown checks wrong table
  ✗ Friend check backwards
  ✗ Log table mismatch
  ✗ Notifications broken
  ✗ Cluster logic wrong
  ✗ No decay system

SEVERITY: 🔴 CRITICAL (Game completely broken)

SOLUTION: Create new RPC (attack_or_claim_territory)
  ✓ Targets territories table
  ✓ Validates speed 2-15 km/h server-side
  ✓ Full energy consumption model
  ✓ Proper protection + shield checks
  ✓ Correct capture logic
  ✓ Uses territory_attack_cooldowns table

IMPLEMENTATION TIME: ~2 hours total
  • Database: 10 min
  • Dart code: 25 min
  • Testing: 20 min
  • Code review: 10 min

───────────────────────────────────────────────────────────────────────────────
USER'S DESIGN INTENT
───────────────────────────────────────────────────────────────────────────────

"Validation of speed countdown and protected will be on app side on the
data from supabase not on the backend so app can easily trigger the conditions"

THIS MEANS:
✓ App validates FIRST (for UX feedback)
  • Check speed, protection, shield instantly
  • Show helpful messages without RPC latency
  • Prevent unnecessary network calls

✓ RPC validates SECOND (for security)
  • Even if app is bypassed/hacked
  • All checks happen server-side
  • Final authority on game state

KEY DATA NEEDED FROM APP:
  • Current speed (from GPS deltas)
  • Territory data with:
    - protection_until (timestamp)
    - shield_until (timestamp)
    - energy (0-60)
  • My attack energy
  • Cooldown status
